# 小说outline纪律：先画图纸再施工

## 核心原则

**outline的质量决定AI写出来的质量。** AI不是作者，它是执行者。给它的规划越模糊，它偏离越远。

## 正确流程

1. **写outline** — 每章都有标题和核心事件描述，关键事件按章节区间分组
2. **写characters** — 每个出场角色都有姓名、身份、性格、作用
3. **写prompt** — 基于outline和characters定制，每本书隔离
4. **逐章写作** — 每写完一章就更新outline的章节记录表（orchestrator自动完成）
5. **卷完结检查** — 30章写完后，检查outline关键事件与实际章节的一致性
6. **反推修正** — 发现脱节的，用实际内容重写outline，同步更新characters

## outline质量标准

### 最低标准（必须满足）

每个关键事件条目格式：
```
N. **章节区间标题**（第X-Y章）：具体事件描述，包含关键情节、人物互动、爽点位置
```

示例（合格）：
```
4. **副本名称**（第11-15章）：规则更复杂，涉及多人合作。主角发现规则之间存在矛盾（规则A说"..."但规则B说"...")，利用矛盾破解。第13章"特定情节"——关键转折发生。
```

示例（不合格）：
```
4. **副本名称**（第11-15章）：规则更复杂，涉及多人合作。主角发现规则之间存在矛盾，利用矛盾破解。配角救了大家。
```

### 卷完结后检查清单

每卷写完30章后，必须执行以下步骤：

1. 提取每章标题和关键实体（人名、地点、物品）
2. 逐条对照outline中的"关键事件"描述
3. 检查项：
   - 事件类型是否相符（outline说"A"，实际写的是"B"？）
   - 关键角色是否出现（outline没提的角色大量出现？）
   - 关键设定是否遗漏（贯穿全文的设定在outline中完全没有？）
   - 情节顺序是否错位（outline说卷末才揭示，实际前面就出现了？）
   - 爽点设计是否相符（outline写"A"，实际写的是"B"？）
4. 发现脱节的，用实际章节内容重写outline的关键事件描述
5. 同步更新characters.md，补充outline中缺失的角色和地点

## 规则类小说特殊要求

规则类小说（规则怪谈、悬疑无限流等）还需要：

1. **规则清单表格** — 每卷必须有完整的规则编号、内容、类型、出现章节
2. **矛盾关系标注** — 明确标注哪些规则之间存在逻辑矛盾
3. **提示真假跟踪** — 每章标注提示是真还是假，跟踪谎言模式

详见 `fanqie-novel-auto` skill 中的规则编号管理部分。

## 为什么这个纪律重要

- AI不是作者，它是执行者。给它的规划越模糊，它偏离越远
- 写完后才发现脱节，意味着后续章节会继续跑偏
- 卷完结后的反推修正是最后补救机会，修完后下一章起才不会继续错
- 每本书的prompt必须隔离，不能跨书引用
