# 写作通用经验与陷阱

## 字数验证

**必须用Python正则统计中文字符，不能用字节数。**

UTF-8编码下，一个中文汉字占3字节。用`wc -c`或`os.path.getsize()`得到的字节数除以3才是大致字符数，但不精确（还有标点、英文等）。

正确方法：
```python
import re
with open(path, 'r', encoding='utf-8') as f:
    content = f.read()
chinese = len(re.findall(r'[\u4e00-\u9fff]', content))
```

排除元数据行（以`【`开头的行）后再统计。

**⚠️ 字数标注必须是真实统计结果，禁止AI估算。** 2026-07-05发现所有5本书的章节末尾字数标注都是AI写的假数字（如"约3100字"实际2708字，"约12500字"实际5819字，book5甚至有"XXXX"占位符）。每章写完后必须立即用Python统计并替换标注。

## 内容扩充策略

### 正确做法
1. write_file写入初稿（约1600-2000字核心情节）
2. Python统计字数
3. 未达标时用`terminal cat >> file << 'EOF'`追加内容（每次约1000-1500字）
4. 再次统计，重复直到达标
5. 最后5字以内时用patch微调

### 追加内容的方向（自然扩充，不显水字数）
- 战斗/训练场景的详细动作描写
- 角色之间的对话扩展（每人多说几句）
- 主角的心理活动和计划分析
- 环境描写和感官细节
- 具体过程的逐步展开

### 不要用patch扩充字数
patch需要精确匹配old_string，长文本匹配成本高，容易陷入死循环。每次patch成功只增加约200-400字，需要10+次才能补足6000字。

## 章节衔接与钩子

每章结尾必须有钩子，且与下一章直接相关。不能在章节末尾用空洞的心灵鸡汤段落代替真正的剧情钩子。

**禁止的结尾模式（模板化）：**
- 同一角色的内心独白反复出现（如"李玄的心中充满了信心"）
- 同一句口号反复出现（如"李家，从今天起，将由他来守护"）
- 同一感慨反复出现（如"修仙世界……也许这就是我的命运吧"）
- 同一动作反复出现（如"他的嘴角微微上扬，露出一丝微笑"）
- 同一结尾语反复出现（如"而这条路，才刚刚开始"）

这些段落每章最多出现一次（通常是第1章）。后续章节必须用具体的剧情悬念作为结尾钩子。

## 每章必须独立

每次cronjob或写作任务只写1章。不要试图在一个任务里写多章。每章有独立的上下文，质量更稳定，不易偏离大纲。

## 写作前校验大纲（强制步骤）

每次写作任务开始前，必须：
1. 读取outline.md，确认当前章节号落在哪个分卷区间
2. 读取该分卷对应的关键事件描述
3. 读取最近2章的实际内容，确认剧情走向
4. 如果大纲要求与已有章节不符，先纠正方向再继续写

## 卷完结后一致性检查

每卷写完30章后，必须：
1. 提取每章标题和关键实体
2. 逐条对照outline中的"关键事件"描述
3. 发现脱节的，用实际章节内容重写outline
4. 同步更新characters.md
5. 更新write_chapter.sh的prompt

## 技术注意事项

- write_file单次输出约1600-2000字中文，超过会截断
- delegate_task的write_file参数不能超过约8K tokens，否则stream超时
- 章节文件命名统一为`chNNN_第X章 标题.md`
- bash八进制陷阱：`ch008`中的`008`会被bash当作八进制报错，需要用`$((10#$num))`转换
- publish_fanqie.py使用`CURRENT_VOLUME_ID`和`CURRENT_VOLUME_NAME`，不是`VOLUME_ID`/`VOLUME_NAME`
- config.env统一标准：MIN_WORDS=2000, MAX_WORDS=3500, CHAPTER_WORDS_TARGET=2500（2026-07-06更新为番茄平台标准字数）
- **delegate_task的goal参数中字数要求必须从config.env动态读取**，不能硬编码。修改config.env后必须检查所有delegate_task调用中的goal是否同步更新。详见 `references/word-count-standard.md`。
