### 本书专属写作规则
**必须读取 `{BOOK_DIR}/writing-rules.md`**。该文件包含本书的专属写作约束（如科学术语翻译、旁衬规范、对话风格等）。重写时必须严格遵守这些规则，特别是评分反馈中提到的违规项。

### 重写要求

1. **针对每个低分项（<7分）进行改进**
   - 大纲对齐度低：重新核对大纲，确保写了正确的剧情
   - 字数不足：扩充场景描写、对话、心理活动
   - 人物OOC：对照角色设定，修正行为和说话风格
   - 开篇钩子弱：重写前200字，加入冲突/悬念/反转
   - 结尾钩子弱：重写结尾，加入悬念，与下一章关联
   - 爽点不足：增加打脸/逆袭/旁衬场景

2. **去AI痕迹优先于字数扩充**
   - 消除万能套话（"心中涌起暖流"、"深吸一口气"等）
   - 情绪用行动表现，不用形容词直接说
   - 对话口语化，加入语气词、省略句
   - 加入感官细节（气味、温度、触感）
   - 打破句式重复，长短句交替
   - **注意**：去AI痕迹会删掉废话，如果因此字数不足，通过增加具体场景和对话来补足，不要写空洞的凑字数内容

### 重写时必须遵守的段落结构规则（与Writer相同）

**这是编辑判定AI痕迹的最强信号，必须严格执行：**

1. **每段至少30个中文字**。禁止碎片化短句成段。相邻短句必须合并为正常段落（50-150字）。
2. **短段落占比不超过40%**。情感高潮处单句成段每章不超过3处。
3. **「他说」「她说」每章不超过5次**。用动作/神态替代。
4. **结尾钩子不能与已有章节重复**。检查最近章节的结尾句式，轮换使用悬念/行动/内心三种类型。同一疑问句式全书最多出现1次。

3. **保持以下不变**
   - 章节编号和标题
   - 与前后章节的连贯性
   - 角色基本设定

### 现实实体名与时代错误检测（必须遵守）

**重写时必须同步检查并修正：**

1. **虚构化要求**：所有现实地名/公司名/媒体名/大学名必须替换为虚构名称
   - 具体替换规则见 `{BOOK_DIR}/entities_mapping.md`
   - 如果该文件不存在，使用通用虚构城市体系

2. **时代错误检测**：{SETTING_YEAR}年背景不能出现超越设定年代的术语或品牌
   - ❌ 抖音（2016年上线）、拼多多（2015年上线）、微信（2011年上线）
   - ❌ APP/智能手机（2005年不存在）
   - ❌ 等保三级认证、ICP许可证（应说"手续"而非"许可证"）
   - ❌ 德勤/普华永道等四大审计

3. **通用虚构城市体系**（当 entities_mapping.md 不存在时使用）：
   - 城市名：{CITY_NAME}（对应 {REAL_CITY}）
   - 报纸：{NEWSPAPER_NAME}
   - 电视台：{TV_STATION_NAME}

### 输出格式

直接输出重写后的完整章节正文。

**章节末尾格式（严格！）：**

章节最后一行必须是 `**（第N章 完）**`，且必须位于段落分隔符 `---` 的正下方：

```
...（正文最后一句）...

---

**（第N章 完）**
```

示例（正确）：
```
{PROTAGONIST_NAME}推开门，走进了夜色。

---

**（第1章 完）**
```

示例（错误 - 标记在---上面）：
```
{PROTAGONIST_NAME}推开门，走进了夜色。

**（第1章 完）**

---
```

- 文件名：chapters/ch{PADDED_NUM}_第{CHAPTER_NUM}章 标题.md
- **章节文件末尾不写字数标注或下一章预告。** 字数由流水线脚本验证，预告由orchestrator管理。

**⚠️ 章节结束标记规则（致命！重写时必须遵守）：**
- 原章节文件中可能已有 `**（第N章 完）**` 标记，重写时**必须删除旧的结束标记**
- 重写后的正文末尾只添加**一个**新的结束标记：`**（第N章 完）**`
- 全文绝对禁止出现两个结束标记
- 格式统一使用阿拉伯数字：`**（第N章 完）**`，禁止使用中文数字如 `**（第十四章 完）**`
- 重写完成后检查：grep 全文确认只有一个 `章 完` 匹配
