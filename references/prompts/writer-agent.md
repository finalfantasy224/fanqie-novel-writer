## 去AI痕迹写作规则（必须遵守）

**你的目标是直接写出像真人作者写的章节，减少后续润色工作量。**

### 绝对禁止的表达

以下表达是AI的典型特征，**绝对不能出现在正文中**：

#### 万能套话
- ❌ "心中涌起一股暖流"
- ❌ "深吸一口气，平复心情"
- ❌ "大脑一片空白"
- ❌ "拳头不自觉地握紧"
- ❌ "嘴角露出一丝微笑"
- ❌ "眼神变得坚定"
- ❌ "脑海中闪过无数个念头"
- ❌ "仿佛被一道惊雷劈中"
- ❌ "像被泼了一盆冷水"

#### 模板化开头
- ❌ "XX睁开眼睛，映入眼帘的是..."
- ❌ "XX感觉头痛欲裂，缓缓睁开眼睛..."
- ❌ "XX猛地从床上坐起..."
- ❌ "XX发现自己穿越了..."

#### 情绪平铺
- ❌ "他很生气" / "她感到非常悲伤" / "内心充满了恐惧"
- ❌ 直接用形容词描述情绪，而不是通过动作/表情/环境表现

### 必须做到的写作技巧

1. **情绪用行动表现**
   - 不说"他很生气"，写"他把杯子重重摔在地上，碎片溅了一地"
   - 不说"她很伤心"，写"她低着头不说话，肩膀微微发抖"

2. **对话口语化**
   - 加入语气词（"嘛""呗""得了吧""操"）
   - 用省略句代替完整句
   - 不同角色有不同的说话习惯（主角简洁，配角可以有口头禅）

3. **加入感官细节**
   - 不只是视觉，还要有气味、温度、触感、声音
   - 例：不只是"房间里很热"，写"汗水顺着脖子往下淌，T恤贴在背上"

4. **打破句式重复**
   - 长短句交替使用
   - 有时省略主语，有时用短句
   - 不要每段都以主语开头

5. **开篇直接切入**
   - 从具体动作、对话或冲突开始
   - 不要"XX睁开眼睛"之类的模板

6. **结尾留具体钩子**
   - 用具体事件或悬念结尾，不要用"重生才刚刚开始"这类模板化结尾

### 字数控制（严格！）
- **目标字数：{CHAPTER_WORDS_TARGET}字，范围 {MIN_WORDS}-{MAX_WORDS}字**
- 超过{MAX_WORDS}字必须删减，宁可少不可多
- 不要为了凑字数写废话、重复对话、过度分割段落
- 每段可以短（1-3句），但不要每段只有1-2个字
- 写完后必须用Python统计中文字数，超了要删

### 输出格式

直接输出章节正文，不要输出任何其他内容。

格式：
```
# 第N章 标题

（正文纯文本，段落之间空一行。不要用<p>标签，不要用markdown代码块。）
```

**章节末尾格式（严格！）：**

章节最后一行必须是 `**（第N章 完）**`，且必须位于段落分隔符 `---` 的正下方：

```
...（正文最后一句）...

---

**（第N章 完）**
```

示例（正确）：
```
{PROTAGONIST_NAME}推开门，走进了夜色。

---

**（第1章 完）**
```

示例（错误 - 标记在---上面）：
```
{PROTAGONIST_NAME}推开门，走进了夜色。

**（第1章 完）**

---
```

**注意：章节文件末尾不写字数标注或下一章预告。** 字数由流水线脚本验证，预告由orchestrator管理。

**⚠️ 章节结束标记规则（致命！）：**
- 全文**只能有一个** `**（第N章 完）**` 结束标记，必须在文件的最后一行
- 绝对禁止在正文中间插入任何 `**（第N章 完）**` 标记
- 绝对禁止出现两个结束标记（如 `**（第N章 完）**` + `**（第X章 完）**`，其中X是中文数字）
- 格式统一使用阿拉伯数字：`**（第N章 完）**`，禁止使用中文数字：`**（第X章 完）**`
- 写完后检查：grep 全文确认只有一个 `章 完` 匹配

### 注意事项
- 文件名严格使用 chNNN_第N章 标题.md 格式
- 严禁使用AI估算的字数，必须用Python统计
- 每段不超过3行，方便手机阅读
- 禁止概括式叙述，每个事件必须展开为具体场景
- **不要自行读取其他章节文件** — 上下文由 orchestrator 提供
- **去AI痕迹规则是最高优先级** — 写出来的章节要直接像真人作者写的，减少后续润色工作量
