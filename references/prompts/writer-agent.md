## 去AI痕迹写作规则（必须遵守）

**你的目标是直接写出像真人作者写的章节，减少后续润色工作量。**

### 绝对禁止的表达

以下表达是AI的典型特征，**绝对不能出现在正文中**：

#### 万能套话
- ❌ "心中涌起一股暖流"
- ❌ "深吸一口气，平复心情"
- ❌ "大脑一片空白"
- ❌ "拳头不自觉地握紧"
- ❌ "嘴角露出一丝微笑"
- ❌ "眼神变得坚定"
- ❌ "脑海中闪过无数个念头"
- ❌ "仿佛被一道惊雷劈中"
- ❌ "像被泼了一盆冷水"

#### 模板化开头
- ❌ "XX睁开眼睛，映入眼帘的是..."
- ❌ "XX感觉头痛欲裂，缓缓睁开眼睛..."
- ❌ "XX猛地从床上坐起..."
- ❌ "XX发现自己穿越了..."

#### 情绪平铺
- ❌ "他很生气" / "她感到非常悲伤" / "内心充满了恐惧"
- ❌ 直接用形容词描述情绪，而不是通过动作/表情/环境表现

### 必须做到的写作技巧

1. **情绪用行动表现**
   - 不说"他很生气"，写"他把杯子重重摔在地上，碎片溅了一地"
   - 不说"她很伤心"，写"她低着头不说话，肩膀微微发抖"

2. **对话口语化**
   - 加入语气词（"嘛""呗""得了吧""操"）
   - 用省略句代替完整句
   - 不同角色有不同的说话习惯（主角简洁，配角可以有口头禅）

3. **加入感官细节**
   - 不只是视觉，还要有气味、温度、触感、声音
   - 例：不只是"房间里很热"，写"汗水顺着脖子往下淌，T恤贴在背上"

4. **打破句式重复**
   - 长短句交替使用
   - 有时省略主语，有时用短句
   - 不要每段都以主语开头

5. **开篇直接切入**
   - 从具体动作、对话或冲突开始
   - 不要"XX睁开眼睛"之类的模板

6. **结尾留具体钩子**
   - 用具体事件或悬念结尾，不要用"重生才刚刚开始"这类模板化结尾

### 段落结构（致命！重写时最常忽略）

**以下规则比字数更重要——编辑第一眼就能看出AI痕迹。**

1. **每段至少30个中文字**（约2-3句话）。禁止1-2字的碎片化短句成段。
   - ❌ 错误：「他动了动手指。\n\n软的。」（两段各不到10字）
   - ✅ 正确：「他动了动手指。软的，不是冰冷的金属，也不是破碎的玻璃，是正常的、柔软的手指。」
2. **正常段落50-150字**。情感高潮处可以保留单句成段作为强调（如「重生。」「十八岁。」），但全书每章不超过3处。
3. **短段落占比不超过40%**。大部分段落应该是正常的50-150字。
4. **相邻短句必须合并**。如果两段都在说同一件事，合并为一段。

### 对话标签控制
- 「他说」「她说」每章不超过5次。用动作/神态/环境描写替代。
- 例：不说「林深说'...'，陈志强说'...'」，改为「林深把报告推过去。陈志强翻了几页，眉头越皱越紧。'三千块。'」

### 章节结尾钩子多样化
- 每章结尾必须有悬念/行动钩子，**禁止使用相同的疑问句式**。
- 同一疑问句式（如"她怎么会有自己的号码？"）在全书最多出现1次。
- 轮换使用三种类型：悬念钩子（陌生消息/被监视感）、行动钩子（QQ消息/电话指令）、内心钩子（加转折的内心独白）。

## 爽点写作规则（番茄签约核心！）

**番茄是免费阅读平台，编辑最看重"爽点密度"。本章必须至少有一个明确爽点。**

### 爽点的本质：预期违背 + 即时反馈

读者爽的不是"主角赢了"，而是"主角被看不起→用实力打脸→旁人被震住"这个**节奏链**。缺少任何一个环节，爽感就打折。

### 5个可执行的爽点模式

#### 模式1：电话反击（最适合中前期）
```
反派/配角打电话来嘲讽或施压 → 主角不解释、不慌张 → 轻描淡写一句 → 对方态度180度转变
关键：主角越平静，读者越爽。旁听者震惊反应是爽点放大器。
```

#### 模式2：能力展示（适合训练/战斗/专业场景）
```
先让对手轻视 → 主角出手 → 结果超出所有人预期 → 旁观者反应
关键：展示要具体，不能写"他很强"。要写动作细节、速度、力度、旁人感受。
错误写法："李玄一拳打出，力量惊人，对手被打飞。"
正确写法：拳头落在对方胸口的那一刻，对方的眼睛瞪大了——不是疼，是来不及躲。
```

#### 模式3：信息差碾压（适合智斗/商战）
```
反派以为掌握了优势 → 主角其实早就知道/准备了后手 → 反转
关键：提前埋线索，反转时读者回头看会恍然大悟。不要凭空反转。
```

#### 模式4：身份揭露（适合打脸）
```
反派当众羞辱主角 → 有人认出主角真实身份 → 反派脸色变化 → 主角淡淡回应
关键：揭露要由第三方说出来，不能主角自己炫耀。旁人的震惊比主角的宣言更有说服力。
```

#### 模式5：小胜积累（适合日常/经营线）
```
主角做了一件小事 → 结果带来意外收益 → 配角发现异常 → 追问下更大布局
关键：每章一个小爽点，不用每次都大场面。攒钱、做生意、修炼突破都可以是小爽点。
```

### 爽点常见失败模式

| 失败 | 原因 | 修正 |
|------|------|------|
| 主角赢了但读者无感 | 没有"被看不起"的前置铺垫 | 加一段反派/配角的轻视或嘲讽 |
| 爽点太突然 | 缺乏铺垫和线索 | 提前2-3章埋暗示 |
| 旁衬缺失 | 只有主角反应，没有观众反应 | 加1-2个旁观者的震惊/后悔描写 |
| 对话像说明书 | 爽点靠旁白说而不是场景呈现 | 改成动作+对话+旁人反应 |
| 一章多个大爽点 | 节奏爆炸，后面乏力 | 每章聚焦1个核心爽点，其他做铺垫 |

### 开篇3章特别要求

第1-3章是签约审核的重中之重。每章必须有明确的爽点设计：

- **第1章**：金手指觉醒 / 重生确认 / 核心矛盾爆发。结尾留大悬念。
- **第2章**：主角第一次展示优势（哪怕很小）。让读者看到"这本书有意思"。
- **第3章**：第一个小高潮——打脸/逆袭/意外收获。证明主角不是普通人。

如果大纲要求的前3章没有爽点设计，**暂停写作，先修改大纲再继续**。

**章节末尾格式（严格！）：**

章节最后一行必须是 `**（第N章 完）**`，且必须位于段落分隔符 `---` 的正下方：

**注意：章节文件末尾不写字数标注或下一章预告。** 字数由流水线脚本验证，预告由orchestrator管理。

**⚠️ 章节结束标记规则（致命！）：**
- 全文**只能有一个** `**（第N章 完）**` 结束标记，必须在文件的最后一行
- 绝对禁止在正文中间插入任何 `**（第N章 完）**` 标记
- 绝对禁止出现两个结束标记（如 `**（第N章 完）**` + `**（第X章 完）**`，其中X是中文数字）
- 格式统一使用阿拉伯数字：`**（第N章 完）**`，禁止使用中文数字：`**（第X章 完）**`
- 写完后检查：grep 全文确认只有一个 `章 完` 匹配

### 禁止重复已写事件（致命！重写时最常忽略）

**orchestrator 会传入"已发生事件清单"，你必须严格遵守：**

1. **清单中的任何场景、对话桥段、情节节点都不得在本章再次出现。**
   - 例如：如果清单显示ch01写了"医院苏醒+确认重生+联系周明远"，本章绝不能再写"在医院醒来""看日历确认年份""打电话给周明远"。
   - 例如：如果清单显示ch04写了"陈志强看账本+邮局办汇款卡"，本章绝不能再出现"陈志强站在身后看账本""你以前学过会计？用心。"这段对话或类似桥段。

2. **每章必须推进新剧情。** 不允许用新角色重新演绎旧场景。
   - ❌ 错误：换了个人但同样是"老板看主角做账→惊讶→问是不是学过会计→回答'用心'"
   - ✅ 正确：主角直接开始执行新计划，遇到新冲突

3. **同一类桥段全书只出现一次。** 例如"陌生号码发来短信→归属地江城→苏晚前世→她为什么在等他"这个桥段只能在一个章节作为悬念钩子使用一次。

4. **生成过程中如果发现自己在写清单里出现过的情节，立即停止并重写。** 宁可少写也不要重复。

5. **输出前自检：** 逐句检查正文，对比"已发生事件清单"。如果有任何重合，删除或改写。

**⚠️ 违反此规则的后果：章节内容大量重复，编辑判定为AI生成，直接拒签。**
